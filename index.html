Migrace z Microsoft Lists

Pokud máte export ze seznamu (ZIP/CSV z vyzkum_*.zip), nejdříve v Supabase SQL Editoru spusťte supabase/kb-records-schema.sql, poté supabase/security-rls.sql.

Import dat z počítače: SUPABASE_SERVICE_ROLE_KEY=<service_role> node scripts/import-list-export.mjs cesta/k/exportu.zip (Service Role klíč: Supabase → Settings → API). V aplikaci pak stačí přihlášení — data se načtou automaticky.

Data a připojení

Hlavní tabulky e-mailů: supabase/kb-records-schema.sql (metadata) + supabase/kb-capture-schema.sql (přílohy, deduplikace). Po přihlášení se data načtou ze Supabase automaticky (autoLoadOnLogin v supabase-config.js).

Načtení e-mailů ze Supabase a správa lokální kopie v prohlížeči.

AI klasifikace

Kontroluji uložený API klíč…

API klíč zůstává pouze v tomto prohlížeči (localStorage). Podporováno OpenAI API a kompatibilní služby.

Zabezpečení

Přihlášení přes Supabase Auth. Data v DB jsou chráněna RLS — spusťte supabase/security-rls.sql. Postup: SECURITY.md.

API klíče (AI, ClickUp) zůstávají jen v tomto prohlížeči. Při odhlášení můžete smazat lokální cache.

Zachytávání znalostí

Pro ukládání nových e-mailů, poznámek a příloh spusťte supabase/kb-capture-schema.sql a supabase/kb-capture-storage.sql, poté supabase/security-rls.sql. Chyba MIME u příloh .msg: spusťte také supabase/kb-capture-storage-migrate-msg.sql.

V aplikaci použijte tlačítko Zachytit znalost (vložení textu, .msg, .eml, hromadný import z Outlook txt, přílohy, AI klasifikace, témata). Pro automatické přeposílání e-mailů nasaďte edge funkci kb-ingest a propojte ji s Power Automate (viz supabase/functions/kb-ingest/index.ts).

Témata v Supabase

Pro trvalá témata spusťte supabase/topics-schema.sql. Pro propojení témat s termíny také supabase/topics-deadlines-migrate.sql.

Termíny v Supabase

Spusťte supabase/deadlines-schema.sql a pokud tabulka už existuje, také supabase/deadlines-migrate-v2.sql.

Import z Excelu: Uložit jako CSV (oddělovač středník) nebo zkopírovat do TSV. Vzor záhlaví: data/deadlines-import.example.tsv.

Osoby v Supabase

Centrální evidence osob: supabase/persons-schema.sql. Propojení s projekty, termíny a e-maily: supabase/persons-links-migrate.sql.

Interní soutěže v Supabase

Connect, Prestige, ReGa a Horizon mají vlastní tabulky: supabase/competitions-program-tables.sql. Migrace ze staré kb_competitions: supabase/competitions-migrate-split-programs.sql. Návraty a PhD Seed zůstávají v legacy competitions-schema.sql.

Prestige sloupce: competitions-migrate-prestige.sql. Connect alokace: competitions-migrate-connect-allocation.sql. PDF: competitions-storage.sql, poté security-rls.sql.

Výzkumné směry PČR

Spusťte supabase/pcr-research-schema.sql (po persons-schema.sql), poté supabase/security-rls.sql nebo jen supabase/pcr-research-rls.sql. Sync z Google Sheets: edge funkce google-sheets-fetch nebo Import CSV.

Rady a orgány

Spusťte supabase/rady-organy-schema.sql. Edge funkce uhk-page-fetch pro stažení stránek uhk.cz.

KB záznamy (AI agent)

Modul čte tabulku kb_items v Supabase (typy TASK, KNOWLEDGE, DECISION, …). Pro úpravu záznamů a export do ClickUp spusťte supabase/kb-items-migrate-v2.sql a přihlaste se. ClickUp token nastavte v Nastavení.

AI poradce

Modul v1 prohledává stávající data (Osoby, Termíny, PČR, soutěže, témata, e-maily, KB záznamy). Pro AI shrnutí použijte stejný API klíč jako u klasifikace e-mailů výše. Uložené dotazy a spojení jsou zatím v prohlížeči; trvalé ukládání: supabase/ai-advisor-schema.sql + security-rls.sql.

Fáze 2 (publikace, aplikované výsledky, hodnotící kritéria, tématizace) rozšíří index znalostí po doplnění tabulek.

EIZ tokeny

Spusťte supabase/eiz-tokens-schema.sql (po persons-schema.sql), na existující DB eiz-tokens-migrate-v2.sql a eiz-tokens-migrate-v3.sql, poté security-rls.sql. Publikace přiřazujte k roku; přehled metrik je po letech.

Databáze časopisů

Spusťte supabase/journal-db-schema.sql, poté security-rls.sql. Import JCR exportů po částech: CSV, TSV, CSC nebo XLSX (Excel). Klíč časopisu = ISSN nebo eISSN. Vzor: data/journal-db-import.example.tsv.
